01 UiButton / 按鈕

Button 採用「基底 + 疊加」設計。先測單一零件，再看專案實際組合。

A. 基底 / 必須
用途 CSS 疊加內容 演示
所有按鈕的最原始基底。任何按鈕都必須先加這個。 .skh-btn 圓角、padding、字重、hover、active、focus、inline-flex、gap。
B. 角色 / Role：只改顏色語意
主要操作：儲存、送出、查詢。 .skh-btn
.skh-btn-primary		 在基底上疊加：藍色背景、白色文字。
次要操作：取消、關閉、返回。 .skh-btn
.skh-btn-secondary		 在基底上疊加：灰藍背景、白色文字。
危險操作：刪除、清空、停用。 .skh-btn
.skh-btn-danger		 在基底上疊加：紅色背景、白色文字。
低干擾操作：首頁、麵包屑、次要返回。 .skh-btn
.skh-btn-ghost		 在基底上疊加：透明背景、藍色文字、hover 淡藍底。
相機 / 掃描專用：GTIN、QR、條碼掃描。 .skh-btn
.skh-btn-camera		 在基底上疊加：紫色背景、白色文字。
C. 尺寸 / Size：只改大小，不改顏色
標準尺寸。這就是 .skh-btn 的原始尺寸。 .skh-btn 基底尺寸：標準 padding、標準字體大小。
小尺寸。表格內編輯、刪除、掃描使用。 .skh-btn
.skh-btn-sm		 在基底上疊加：padding 變小、字體變小。
大尺寸。手機主操作、簽到送出使用。 .skh-btn
.skh-btn-lg		 在基底上疊加：padding 變大、字體變大。
滿版寬度。手機主操作或單一重要操作使用。 .skh-btn
.skh-btn-full		 在基底上疊加：寬度變成 100%。
D. 形狀 / Shape：只改外型，不改顏色
一般圓角。這就是 .skh-btn 的原始形狀。 .skh-btn 使用基底的 border-radius: var(--skh-radius)。
純 icon 按鈕。表格操作、掃描、刪除常用。 .skh-btn
.skh-btn-icon		 在基底上疊加：固定寬高、移除 padding，讓 icon 置中。
膠囊型。分類、狀態切換、標籤式按鈕使用。 .skh-btn
.skh-btn-pill		 在基底上疊加：圓角變成 999px。
E. 狀態 / State：只改互動狀態
正常可點擊狀態。這就是基底按鈕的預設互動。 .skh-btn 基底提供 hover 上浮、active 回彈、focus 外框。
載入中。Apps Script 等待、儲存中、防止重複點擊。 .skh-btn
.skh-btn-loading		 在基底上疊加：左側 spinner、透明度下降、禁止點擊。
已選取。Toggle、職級、健保/自費切換使用。 .skh-btn
.skh-btn-selected		 在基底上疊加：選取背景、選取文字色、選取邊框。
禁用。不可操作，也不應該 hover 上浮。 .skh-btn
disabled		 在基底上疊加：透明度下降、游標禁止、取消陰影與位移、禁止點擊。
F. 群組 / Group：只負責排列，不負責顏色
按鈕群組。多個同類選項並排，例如健保/自費。 .skh-btn-group 提供 inline-flex、gap、wrap。它不提供顏色，顏色由子按鈕決定。
工具列。後台、建檔、匯出、排序常用。 .skh-btn-toolbar 提供 flex、gap、wrap。它不提供顏色，顏色由子按鈕決定。
