# 敷料建檔 Table Regression Checklist

版本：Table-State Foundation  
用途：每次修改敷料建檔前端後，merge 前至少跑「核心測試」；重大修改再跑「完整測試」與「最終壓力測試」。

---

## 0. Merge 前基本原則

- [ ] 目前在測試分支，不是在 master 直接改
- [ ] 最新前端 HTML/TXT 已貼上並部署測試版
- [ ] 沒有明顯 console error
- [ ] 主要流程實測通過後，才 merge 回 master
- [ ] 每修掉一個 bug，都要把對應測項加進本 checklist

---

## 1. 載入與初始狀態

- [ ] 開啟敷料建檔頁
- [ ] 主檔正常讀取
- [ ] 掃描按鈕可按
- [ ] 更改排序按鈕可按
- [ ] 批次儲存按鈕一開始不顯示
- [ ] 未按編輯時，單包 GTIN / 包裝 GTIN 欄內小相機不顯示

---

## 2. 新增流程

- [ ] 掃描不存在 GTIN
- [ ] 新列出現在最上方
- [ ] 新列進入編輯狀態
- [ ] 新列顯示「儲存」與「不儲存」
- [ ] 新列可補院內碼、名稱、規格、價格、狀態
- [ ] 新列儲存成功
- [ ] 儲存後離開 frontOnly / 新增狀態
- [ ] 儲存後重新掃描同 GTIN 可找到該資料

---

## 3. 不儲存 / 前端暫存列

- [ ] 新增一列後按「不儲存」，該列消失
- [ ] 新增兩個未儲存列，刪除其中一個，另一個仍保留
- [ ] 不儲存不會 reload 整張表
- [ ] 不儲存不會清掉其他未儲存列
- [ ] 不儲存後再掃同一 GTIN，會重新開始新增流程
- [ ] 不儲存後不會跳回舊的暫存列
- [ ] 不儲存後 scan memory 有被清掉

---

## 4. 掃描取消流程

- [ ] 掃描不存在 GTIN 後進入新增流程
- [ ] 中途取消流程
- [ ] 暫存列被移除
- [ ] 不 reload 整張表
- [ ] 再掃同一 GTIN，不會回到舊列
- [ ] 再掃同一 GTIN，會重新建立新流程

---

## 5. 編輯既有列

- [ ] 按「編輯」後欄位可修改
- [ ] 按「編輯」後 GTIN 欄內小相機出現
- [ ] 修改任一欄位後，主按鈕變成「儲存」
- [ ] 沒有修改時，主按鈕可返回
- [ ] 取消編輯會恢復原本資料
- [ ] 儲存既有列成功
- [ ] 儲存後離開編輯模式
- [ ] 儲存後 table mode 不會卡在 saving
- [ ] 儲存後其他列不會永久 disabled

---

## 6. 編輯中仍可掃描

- [ ] 編輯既有列時，上方主掃描仍可用
- [ ] 編輯既有列時，列內小相機仍可用
- [ ] 編輯既有列時，可繼續掃描新增另一列
- [ ] 編輯既有列時，批次儲存判斷正常
- [ ] 編輯既有列時，更改排序按鈕不可用

---

## 7. 新增 + 編輯混合狀態

- [ ] 新增列存在時，可編輯既有列
- [ ] 編輯既有列存在時，可新增新列
- [ ] 新增列 + 編輯列同時存在時，儲存新增列不會洗掉編輯列
- [ ] 新增列 + 編輯列同時存在時，儲存編輯列不會洗掉新增列
- [ ] 新增列 + 編輯列同時存在時，取消編輯不會洗掉新增列
- [ ] 新增列 + 編輯列同時存在時，不儲存新增列不會影響編輯列
- [ ] 任一單列儲存後，表格狀態會回到 adding / editing / clean，不會停在 saving
- [ ] 有未儲存或編輯列時，單筆操作不應強制整表 reload

---

## 8. 批次儲存

- [ ] 同時修改兩列時，批次儲存按鈕出現
- [ ] 只修改一列時，批次儲存按鈕不出現
- [ ] 批次儲存可成功儲存多列
- [ ] 批次儲存後所有成功列離開編輯狀態
- [ ] 批次儲存後沒有卡在 saving
- [ ] 批次儲存後 scan memory / cache 狀態正常
- [ ] 批次儲存後重新掃描已儲存 GTIN 可找到資料

---

## 9. 刪除流程

- [ ] 已儲存資料按刪除會詢問確認
- [ ] 取消刪除後資料仍在
- [ ] 確認刪除後資料移除
- [ ] 刪除已儲存資料後，再掃同 GTIN 不會復活舊資料
- [ ] 有未儲存新增列時，刪除既有列不會洗掉未儲存列
- [ ] 有編輯列時，刪除 / 取消編輯不會讓其他列卡死

---

## 10. Cache / Scan Memory

- [ ] 新增後掃描同 GTIN 找到新資料
- [ ] 修改 GTIN 後，舊 GTIN 不應誤指到新列
- [ ] 修改後掃描可找到修改後資料
- [ ] 刪除後掃描不會貼回舊資料
- [ ] 取消掃描後 scan memory 清除
- [ ] 不儲存後 scan memory 清除
- [ ] 再掃同碼不會跳回已取消或已刪除的暫存列

---

## 11. 排序流程

- [ ] 沒有編輯列 / 新增列時可進入排序
- [ ] 有編輯列時不能進入排序
- [ ] 有未儲存新增列時不能進入排序
- [ ] 排序模式下不能編輯
- [ ] 排序模式下不能刪除
- [ ] 排序模式下不能掃描
- [ ] 儲存排序成功
- [ ] 儲存排序後重新讀取順序正確

---

## 12. 離開保護

- [ ] 有未儲存新增列時，按「開始領用」會跳提示
- [ ] 有編輯中列時，按「開始領用」會跳提示
- [ ] 有未儲存資料時，重新開啟敷料建檔 / 重新讀取會跳提示
- [ ] 有未儲存資料時，瀏覽器刷新頁面會出現離開提示
- [ ] 沒有未儲存資料時，可正常切換頁面

---

## 13. GTIN / 條碼解析

- [ ] 13 碼 GTIN 可查
- [ ] 14 碼 GTIN 可查
- [ ] 前面補 0 視為同一碼
- [ ] GS1 條碼可解析
- [ ] 傳統條碼可解析
- [ ] 單包 GTIN 與包裝 GTIN 不互相誤判
- [ ] 包裝 GTIN 可帶出正確一盒數量

---

## 14. UI 狀態

- [ ] 未編輯列的小相機隱藏
- [ ] 編輯列的小相機顯示
- [ ] saving / deleting / loading 時按鈕暫時 disabled
- [ ] 操作完成後按鈕恢復
- [ ] 批次儲存顯示 / 隱藏時機正確
- [ ] 通知不會遮住主要操作
- [ ] 手機版工具列可水平滑動，不擠壓表格

---

## 15. 最終壓力測試建議

這組是 merge 前最重要的整合測試。

### 壓力測試 A：新增 + 新增 + 不儲存

- [ ] 掃描新增 A
- [ ] 掃描新增 B
- [ ] A 按「不儲存」
- [ ] B 仍保留
- [ ] 再掃 A，不會跳回舊暫存列
- [ ] B 可正常儲存

### 壓力測試 B：新增 + 編輯 + 儲存新增

- [ ] 掃描新增 A
- [ ] 編輯既有列 B
- [ ] 儲存新增 A
- [ ] B 仍在編輯狀態
- [ ] B 沒有被刷新洗掉
- [ ] B 可正常儲存或取消

### 壓力測試 C：新增 + 編輯 + 儲存編輯

- [ ] 掃描新增 A
- [ ] 編輯既有列 B
- [ ] 儲存 B
- [ ] A 仍保留
- [ ] A 沒有被刷新洗掉
- [ ] A 可正常儲存或不儲存

### 壓力測試 D：取消與再掃描

- [ ] 掃描不存在 GTIN A
- [ ] 中途取消
- [ ] 再掃 A
- [ ] 重新開始新增流程
- [ ] 不會跳回舊 key / 舊暫存列

### 壓力測試 E：完整混合流程

- [ ] 掃描新增 A
- [ ] 掃描新增 B
- [ ] 編輯既有列 C
- [ ] 儲存 A
- [ ] 不儲存 B
- [ ] 儲存 C
- [ ] 確認沒有任何列卡在「儲存中」
- [ ] 確認小相機只在編輯列顯示
- [ ] 確認沒有未儲存列後可排序
- [ ] 儲存排序
- [ ] 重新讀取主檔
- [ ] 確認資料仍正確

---

## 16. Merge 條件

符合以下條件才 merge 回 master：

- [ ] 核心流程全過
- [ ] 新增 + 編輯混合狀態全過
- [ ] 不儲存 / 取消掃描不會殘留記憶
- [ ] 儲存後不會卡在 saving
- [ ] 最終壓力測試至少 A、B、C、D 通過
- [ ] 沒有 blocking bug
- [ ] checklist 已同步更新
- [ ] HTML/TXT 已存入 repo
- [ ] checklist 已存入 repo
